# Store API

## Responsibilities

- Read store catalog
- Purchase items
- Read item detail

## Current contract

The current web app already contains a store route and store data loading pattern.
This document defines the API contract for the database-backed sprint.

## Reserved endpoint categories

- Store catalog
- Item detail
- Purchase request
- Purchase confirmation
- Purchase history

## Rules

- Prices are configuration-driven.
- Purchase effects are server-authoritative.
- Store must not introduce pay-to-win rules.
