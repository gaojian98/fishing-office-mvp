# Milestone 1 Founder Test Guide

## 测试入口

- 主入口：`/office-fishing/`
- 备用入口：`/office-fishing.html`

## 测试目标

确认当前 FishingOffice MVP 可以在手机浏览器上稳定试玩，不要求新增功能。

## 手机测试步骤

### 1. 打开首页

预期结果：

- 页面正常打开。
- Home 背景完整显示，不裁剪。
- 顶部、底部按钮可见。
- Debug Panel 仅在 Debug 环境可见。

### 2. 执行抛线

预期结果：

- 点击抛线相关热区后进入等待状态。
- 页面显示 WaitingEvents。
- 当前鱼饵信息正确更新。

### 3. 等待并查看事件

预期结果：

- 页面出现等待事件列表。
- 不会卡死。
- 不会出现报错弹窗。

### 4. 收线

预期结果：

- 点击收线后生成 FishResult。
- 弹出结果处理界面。
- 不会出现状态死锁。

### 5. 出售鱼

预期结果：

- Wallet 中 `fish_coin` 增加。
- Transaction 增加一条出售记录。
- 页面回到可继续操作状态。

### 6. 再次钓鱼并作为鱼饵

预期结果：

- 当前结果鱼可作为下一次鱼饵。
- 下一次结果鱼链提升。
- 页面状态正常切换。

### 7. 留下鱼

预期结果：

- Inventory 中增加鱼。
- 空库存状态不会报错。

### 8. 检查常用入口

预期结果：

- Store 可打开。
- Wallet 可打开。
- Inventory 可打开。
- Dialog 可正常关闭。

## 如何记录 Bug

1. 记录页面名称。
2. 记录操作步骤。
3. 记录实际结果。
4. 记录期望结果。
5. 附上截图。
6. 标明严重程度。

## 严重程度建议

- `P0`：无法启动、白屏、崩溃、数据丢失
- `P1`：核心流程被阻断
- `P2`：流程可用但有明显体验或逻辑问题
- `P3`：文案、对齐、细节问题
